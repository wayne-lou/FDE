# Project Overview

RaceOps AI is a one-FDE realistic demo for racing team operations. It avoids deep race engineering data and instead targets a lower-friction, high-value area: driver rest, recovery, logistics, schedule pressure, meeting load, and media overload.

The goal is to show FDE value: enter a live operational environment, understand messy workflows, integrate existing data, and ship a production-shaped AI workflow quickly.

## v2 FDE-style Three.js Digital Twin
The Three.js panel is no longer decorative. It represents a race-weekend operations room with business objects:
- Garage, Engineering, Media, Recovery, Hotel, and Transport zones.
- Driver risk markers mapped to fatigue assessment records.
- Agent workflow routes that change after a natural-language request, showing how the AI recommendation moves work across operations resources.

This keeps the demo lightweight enough for a single FDE while still showing the FDE value: turn a vague operational problem into a visible, human-reviewed workflow.
